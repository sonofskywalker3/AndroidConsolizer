# Android Consolizer 3.6.0 — Bug Fix Release

A round of fixes responding to Nexus user reports against 3.5, plus one bonus quality-of-life feature for furniture placement.

## Bug fixes

**Beds no longer bounce back to inventory after placement.** The most-reported issue from 3.5 is fixed. Place a bed and it stays placed.

**Right thumbstick no longer drifts the cursor in the overworld.** Vanilla Android maps the right stick to mouse motion, which made interact and sickle target the wrong tile. New GMCM toggle to suppress it (on by default).

**Dresser no longer destroys clothes.** The shipping-bin handler was wrongly catching dresser interactions and selling clothing instead of storing it. Storage shop interactions are now isolated.

**Aquarium no longer duplicates fish on take.** Y-button take-one now properly removes the fish from the tank.

**Quest Log accessible via Hold Start.** Tap Start opens the game menu (as before); hold Start for ~½ second to open the Quest Log/Journal. Works on Xbox Bluetooth controllers where the Back/Select button doesn't register.

**Storage shop polish.** Dresser and aquarium navigation, sell-tab item highlighting, deposit restrictions, hover refresh, and default buy-tab selection all match console/PC behaviour.

**Other fixes**
- Chest deposits now fire bundle/quest hooks via `behaviorFunction`.
- Equipment slot equip/unequip routes through the game's own `Farmer.Equip` so set-bonus callbacks fire.
- Shop stock and post-purchase actions match vanilla.
- Fishing rod tackle attaches correctly via controller (trap bobber, magnet, etc.).
- Auto-Grabber no longer accepts console-style A/Y deposits (which broke its grab contract).

## Bonus feature: Console Furniture Placement

The vanilla Android placement preview shows a multi-tile green/red map covering every tile where the furniture's *top-left corner* can land. For multi-tile pieces like beds, this is misleading — the green tiles cluster above and to the left of where the furniture will actually sit, so aiming at the bed's centre shows only red.

3.6.0 replaces this with a console-style preview: a single ghost rectangle showing exactly where the furniture will land, plus a translucent sprite of the furniture itself rendered on top. You see what you're placing and where, with the validity highlight showing through the translucent sprite.

New GMCM toggle: **Console Furniture Placement** (on by default). Disable it if you prefer the vanilla multi-tile map.

## Compatibility & toggles

Every fix is a separate GMCM toggle. If something doesn't suit your setup or conflicts with another mod, you can disable it individually. See the GMCM config menu in-game.
